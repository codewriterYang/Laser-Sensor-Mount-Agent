# Route D 参考图方案对比

**日期**: 2026-06-13
**目标**: 对比四种参考图生成方案，选择最适合 Seedream img2img 的方案

---

## 方案概览

```
方案 A: Wireframe Render      — 纯线框（边线绘制）
方案 B: Hidden Line Render     — 隐藏线移除（仅可见边线）
方案 C: Grey CAD Render        — 灰色面片渲染（带简单光照）
方案 D: Simple Mesh Render     — 三角网格渲染（完整光照）
```

---

## 详细对比

### 方案 A: Wireframe Render

**原理**: 从 STEP 文件中提取 EDGE_CURVE、LINE、CIRCLE 实体，投影到 2D，用 Pillow 绘制线条。

```
STEP → 正则提取边线 → 3D→2D 投影 → Pillow draw.line() → PNG
```

| 维度 | 评估 |
|---|---|
| **实现成本** | 2-3 天 |
| **Python 3.13 兼容** | ✓ 完全兼容（纯 Python + Pillow） |
| **新依赖** | 无 |
| **几何保真度** | ★★☆☆☆ — 仅边线，无面信息 |
| **视觉质量** | 线框图，所有边可见（含隐藏线） |
| **对 AI 辅助效果** | ★★★☆☆ — AI 能理解轮廓但缺乏深度信息 |

**优点**:
- 零新依赖
- 实现简单
- 能表达零件的基本轮廓

**缺点**:
- 所有边线都可见（含被遮挡的），视觉混乱
- 无面信息，无法区分正面/背面
- 复杂零件（766 面）的线框会非常密集

**STEP 数据使用率**: ~5%（仅使用 EDGE_CURVE 的端点坐标）

---

### 方案 B: Hidden Line Render

**原理**: 从 STEP 拓扑关系计算面的遮挡关系，仅绘制可见边线。

```
STEP → 解析 ADVANCED_FACE + EDGE_LOOP → 计算遮挡 → 仅绘制可见边 → PNG
```

| 维度 | 评估 |
|---|---|
| **实现成本** | 5-7 天 |
| **Python 3.13 兼容** | ✓ 可兼容（需自实现遮挡算法） |
| **新依赖** | 无（但需要 numpy 加速） |
| **几何保真度** | ★★★☆☆ — 仅边线但仅可见部分 |
| **视觉质量** | 工程图风格，清晰干净 |
| **对 AI 辅助效果** | ★★★☆☆ — 清晰但无深度/材质信息 |

**优点**:
- 工程图标准风格
- 无视觉混乱（隐藏线已移除）
- 适合 AI 理解

**缺点**:
- 实现复杂（需要面的法向量计算、遮挡判断）
- 正则解析 STEP 拓扑关系困难
- 无面信息，仍然是"线框"

**STEP 数据使用率**: ~15%（使用 EDGE_CURVE + ADVANCED_FACE 拓扑）

---

### 方案 C: Grey CAD Render

**原理**: 从 STEP 解析 ADVANCED_FACE 的面片数据，三角化后用简单光照渲染。

```
STEP → 解析面片 → 三角化 → 法向量计算 → 光照渲染 → Pillow 像素填充 → PNG
```

| 维度 | 评估 |
|---|---|
| **实现成本** | 5-7 天 |
| **Python 3.13 兼容** | ⚠️ 需验证（三角化算法复杂） |
| **新依赖** | numpy（可选 scipy） |
| **几何保真度** | ★★★★☆ — 有面、有深度、有光照 |
| **视觉质量** | 灰色 CAD 渲染，有立体感 |
| **对 AI 辅助效果** | ★★★★☆ — AI 能理解形状、深度、材质位置 |

**优点**:
- 有面信息，能区分正面/背面
- 有简单光照，有立体感
- 作为参考图效果最好

**缺点**:
- 实现复杂（需要三角化 + 光栅化 + 深度缓冲）
- 正则解析 B 样条曲面几乎不可能
- 平面可以用正则解析，但曲面需要数值计算

**STEP 数据使用率**: ~25%（使用 ADVANCED_FACE + 曲面参数）

---

### 方案 D: Simple Mesh Render

**原理**: 用 trimesh/opencascade 将 STEP 三角化为 mesh，然后软件渲染。

```
STEP → OCP STEPControl_Reader → BRepMesh → 三角面片 → 光栅化 → PNG
```

| 维度 | 评估 |
|---|---|
| **实现成本** | 3-5 天（如果 OCP 安装成功） |
| **Python 3.13 兼容** | ✗ OCP 主要支持 Python 3.10-3.12 |
| **新依赖** | OCP (~150MB) + numpy |
| **几何保真度** | ★★★★★ — 完整三角 mesh |
| **视觉质量** | 接近专业 CAD 渲染 |
| **对 AI 辅助效果** | ★★★★★ — 最准确的几何参考 |

**优点**:
- 几何保真度最高
- 三角化由 OpenCASCADE 工业级内核完成
- 输出 mesh 可复用

**缺点**:
- OCP 不支持 Python 3.13
- 需要 conda 环境或降级 Python
- 安装体积大 (~150MB)
- 与 venv 集成困难

**STEP 数据使用率**: ~90%（使用完整 B-Rep 几何）

## 对 Seedream img2img 的适配度

Seedream img2img 的工作原理：
- 参考图提供**几何结构**（形状轮廓、比例、位置）
- Prompt 提供**材质/光照/风格**描述
- strength 控制两者之间的平衡

**关键洞察**: Seedream 不需要参考图是"高质量"的——它需要参考图是**结构准确**的。

| 方案 | 结构准确度 | 作为参考图效果 |
|---|---|---|
| A: Wireframe | 低（所有边可见） | ★★☆☆☆ — AI 可能被混乱线框误导 |
| B: Hidden Line | 中（仅可见边） | ★★★☆☆ — 清晰但无深度 |
| C: Grey CAD | 高（有面+光照） | ★★★★☆ — AI 能理解 3D 结构 |
| D: Mesh Render | 最高（完整 mesh） | ★★★★★ — 最准确的参考 |

## 关键结论

1. **方案 A (Wireframe) 和方案 D (Mesh) 总分并列最高**，但原因不同
2. 方案 A 赢在实现成本和兼容性，但保真度低
3. 方案 D 赢在保真度和 AI 效果，但 Python 3.13 不兼容
4. **方案 C (Grey CAD) 是最佳平衡点** — 保真度和 AI 效果都较好
5. 但方案 C 的正则解析曲面是技术难点

**实际可行路径**: 方案 A（Wireframe）可以在 2-3 天内完成，配合 Seedream img2img 已经能显著提升效果。方案 C/D 可作为后续优化。
